**OPC 2026 数字文化赛道 · Nova**

## 1. 产品定位

LearnMate 是一个面向长期备考人群的 AI 学习工作台。它不是另一个内容生成工具，而是把用户每天的学习过程组织成**可执行的闭环**——从情绪启动、苏格拉底式追问、资料解析到间隔复习，让 AI 真正成为学习流程的搭子。

**核心主张**：不堆内容，只组织流程。

---

## 2. 应用场景

### 2.1 目标用户画像

| 用户类型 | 典型场景 | 痛点 |
|---------|---------|------|
| 雅思备考者 | 每天需要练习写作和口语，但缺乏反馈 | 不知道自己的答案好不好，如何改进 |
| 考研学生 | 大量资料需要消化吸收 | 看了就忘，无法形成系统知识 |
| 公考/资格证考生 | 备考周期长，容易中途放弃 | 焦虑、摆烂、卡住导致学习中断 |

### 2.2 核心使用场景

#### 场景一：每日学习启动（3 分钟）
1. 用户打开 LearnMate，看到今日 3 个任务
2. 选择一个挑战任务，写下自己的理解
3. Ask Agent 通过苏格拉底式追问推动深度思考
4. 完成今日第一个学习闭环

#### 场景二：资料解析（10 分钟）
1. 用户粘贴一篇雅思范文或教材段落
2. 系统自动提取知识节点（概念、结构、表达）
3. 生成针对性练习题和间隔复习计划
4. 知识节点自动加入个人知识图谱

#### 场景三：复习与错题分析（5 分钟）
1. Mem Agent 根据遗忘曲线提醒复习
2. Data Agent 分析错题模式，定位薄弱点
3. 用户完成复习任务，更新知识掌握度

## 3. 核心功能

### 3.1 今日学习闭环

**功能描述**：系统每天给出 3 个以内的任务，确保用户能在短时间内完成一个完整的学习循环。

**任务类型**：
- 今日挑战（苏格拉底式追问）
- 到期复习（间隔重复）
- 错题分析（弱点定位）

**成功标准**：用户每天至少完成 1 个任务

### 3.2 多 Agent 协作

| Agent | 角色定位 | 核心能力 |
|-------|---------|---------|
| Star | 情绪陪伴 | 降低启动成本，接住焦虑和摆烂情绪 |
| Ask | 追问导师 | 苏格拉底式提问，一次只问一个问题，推动深度理解 |
| Mem | 记忆管家 | 根据遗忘曲线安排间隔复习 |
| Data | 学习分析师 | 定位错题模式，分析薄弱领域 |

**协作流程**：
```
用户输入 → Star 接住情绪 → Ask 追问理解 → Mem 安排复习 → Data 分析弱点
```

### 3.3 资料到知识节点

**输入**：用户粘贴学习材料（文章、笔记、教材段落）

**处理**：
1. 提取核心概念
2. 生成结构化知识卡片（标题、摘要、领域）
3. 生成针对性练习题
4. 安排间隔复习计划（D1/D3/D7）

**输出**：
- 知识节点卡片
- 练习题
- 复习安排
- 知识图谱更新

### 3.4 个人知识图谱

**可视化展示**：
- 节点 = 知识概念（来自用户材料）
- 连线 = AI 发现的概念关系
- 节点大小 = 掌握程度
- 节点颜色 = 领域分类

**更新机制**：每次资料解析自动添加新节点，复习记录更新节点状态

## 4. 评测标准

### 4.1 产品可用性评测

| 指标 | 目标值 | 测量方法 | 权重 |
|------|--------|---------|------|
| 首次上手时间 | ≤90 秒 | 从打开到完成第一次挑战或资料解析 | 20% |
| 每日任务完成率 | ≥1 个/天 | 统计用户每天完成的任务数 | 25% |
| 知识节点准确性 | ≥4/5 分 | 人工抽检节点质量（相关性、清晰度） | 20% |
| 追问质量 | 推动理解 | Ask 是否一次只问一个问题，是否引导用户自己思考 | 20% |
| 闭环完整度 | 4 项齐全 | 资料解析后是否生成：节点+练习+复习+图谱 | 15% |

### 4.2 技术降级评测

| 场景 | 降级策略 | 可用性 |
|------|---------|--------|
| 无外部 API | 使用本地规则引擎 | 可完成完整演示闭环 |
| 网络不稳定 | 本地状态缓存 | 数据不丢失 |
| 移动端访问 | 响应式布局 | 核心功能可用 |

### 4.3 评委演示路径

**推荐演示流程（3 分钟）**：

1. **打开静态页面**（`open index.html`）
2. **今日页**：提交一个挑战回答，看 Ask 追问反馈
3. **搭子页**：切换 Star/Ask/Mem/Data，体验多角色协作
4. **资料页**：粘贴雅思写作材料，生成知识节点和练习题
5. **图谱页**：查看知识如何连成网络

**成功标志**：评委在 90 秒内完成一次完整的资料解析流程

## 5. MVP 边界

### 5.1 本版本包含

✅ 静态交互演示（HTML/CSS/JS）  
✅ Gradio 多 Tab 演示  
✅ 4 个 Agent 角色对话  
✅ 资料解析生成知识节点  
✅ 知识图谱可视化  
✅ 间隔复习计划生成  

### 5.2 本版本不包含

❌ 真实账号系统  
❌ 外部 API 接入（OpenAI 等）  
❌ 题库系统  
❌ 支付/课程系统  
❌ 云同步  
❌ 移动端 App  

### 5.3 后续迭代计划

**V2（复赛阶段）**：
- 接入真实 LLM API
- 用户账号系统
- 错题本功能
- 学习数据统计

**V3（决赛阶段）**：
- 移动端适配
- 社区学习小组
- 学习报告导出
- 个性化推荐引擎

## 6. 竞争优势

| 维度 | 传统学习工具 | LearnMate |
|------|------------|-----------|
| 内容 vs 流程 | 堆内容 | 组织流程 |
| AI 角色 | 问答工具 | 学习搭子 |
| 学习闭环 | 用户自己组织 | 系统自动闭环 |
| 情绪支持 | 无 | Star Agent 接住焦虑 |
| 知识管理 | 孤立笔记 | 关联图谱 |

## 7. 技术架构

  ↓
